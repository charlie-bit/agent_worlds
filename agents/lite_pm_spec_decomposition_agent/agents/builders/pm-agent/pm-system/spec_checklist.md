# Spec Checklist

Before implementation starts, check:

- [ ] The one-liner is clear.
- [ ] The target platform is defined.
- [ ] The target user is defined or intentionally assumed.
- [ ] The MVP goal is testable.
- [ ] The core loop is written as a flow.
- [ ] `P0`, `P1`, and `Later` are separated.
- [ ] Every `P0` item has acceptance criteria.
- [ ] Out of scope is explicit.
- [ ] Blocking open questions are listed.
- [ ] Assumptions are visible.
- [ ] No `Later` feature is required to validate the MVP.
